LabVIEW VI Props — Prototype

本 prototype 从 labview-vi-props-skill 中提炼了核心经验，实现了一组跨版本稳定读取 LabVIEW VI 属性的独立脚本。

文件结构

prototype/
└── scripts/
    ├── discover_installed_labviews.py   # 独立探测本机 LabVIEW 安装（注册表 + PE 头）
    ├── read_vi_version.py                # 读取 VI 保存版本（纯文件头，无需 LabVIEW）
    ├── read_vi_description.py            # 读取 VI Description（通过 COM/ActiveX）
    ├── read_vi_props.py                  # 读取 VI 全部可访问属性（通过 COM/ActiveX）
    ├── read_vi_icon.py                   # 直接解析 RSRC 导出 VI 图标（纯文件）
    └── save_vi_panel_image.py            # 通过 Python + VI Server 导出前/背面板图片
注意：VBScript COM Worker（.vbs）的正式版本位于仓库根目录的 workers/ 目录下，由扩展运行时直接调用。 prototype/scripts/ 中不再保留 .vbs 副本，以避免调试时指向过时版本。

discover_installed_labviews.py — 探测本机 LabVIEW 安装

这是一个独立的诊断 prototype。它不依赖 VS Code 扩展宿主，专门用于回答一件事： 这台 Windows 机器上，注册表里到底有哪些 LabVIEW 安装能被可靠识别出来。

工作内容

  1. 扫描 64 位与 32 位注册表视图下的 HKLM\SOFTWARE\National Instruments\LabVIEW。
  2. 读取每个版本子键的 Path。
  3. 检查该目录下是否存在 LabVIEW.exe。
  4. 读取 LabVIEW.exe 的 PE 头，判断 x86 / x64；必要时回退到按安装目录推断位数。
  5. 输出有效安装列表，并可选输出所有被跳过条目与原因。

用法

# 只看有效安装
python scripts/discover_installed_labviews.py

# 附带所有被跳过条目的诊断信息
python scripts/discover_installed_labviews.py --diagnose

# JSON 输出（适合和扩展运行时结果对比）
python scripts/discover_installed_labviews.py --json

# JSON + 完整诊断
python scripts/discover_installed_labviews.py --json --diagnose

典型输出

发现 7 个可用 LabVIEW 安装：
- LabVIEW 2017 32bit | key=17.0 | view=32 | dir=C:\Program Files (x86)\National Instruments\LabVIEW 2017
- LabVIEW 2020 64bit | key=20.0 | view=64 | dir=C:\Program Files\National Instruments\LabVIEW 2020
- LabVIEW 2025 SP3 32bit | key=25.3 | view=32 | dir=C:\Program Files (x86)\National Instruments\LabVIEW 2025

退出码

退出码含义
0脚本执行成功，且至少发现一个可用安装
2没有发现可用安装，或探测执行失败；详细错误会打印到 stderr

read_vi_version.py — 读取 VI 版本

不依赖 LabVIEW 安装，纯文件头解析，跨平台可用。 适合在没有 LabVIEW 的环境（CI、服务器）中快速批量检查 VI 版本。

用法

# 单文件
python scripts/read_vi_version.py "C:\path\to\test.vi"
# 输出: 17.0

# 完整年份格式
python scripts/read_vi_version.py --year "C:\path\to\test.vi"
# 输出: 2017.0

# 人类可读描述
python scripts/read_vi_version.py --describe "C:\path\to\test.vi"
# 输出: LabVIEW 2017 SP0

# 批量扫描目录（递归）
python scripts/read_vi_version.py --recursive --year "C:\project\src"

# JSON 输出（适合程序消费）
python scripts/read_vi_version.py --json --year "C:\path\to\test.vi"

版本号解码规则

VI 文件头前 512 字节内有 00 00 00 A0 标记，紧随其后两个字节以 BCD 格式编码版本：

字节偏移 +4：主版本 BCD  0x17 → 23（十进制）→ 2017 年
字节偏移 +5：次版本 BCD  0x00 → 0 → SP0，0x03 → 3 → SP3
BCD 字节值十进制主版本完整年份LabVIEW 名称
0x14142014LabVIEW 2014
0x17172017LabVIEW 2017
0x20202020LabVIEW 2020
0x25252025LabVIEW 2025

Python API

from scripts.read_vi_version import read_vi_version, describe_version, major_to_year

version = read_vi_version(r"C:\path\to\test.vi")
# 返回 (17, 0) 或 None（非有效 VI 文件）

if version:
    major, minor = version
    print(major_to_year(major))       # 2017
    print(describe_version(major, minor))  # LabVIEW 2017 SP0

退出码

退出码含义
0全部成功
2文件不存在
3非有效 VI 文件或读取失败

read_vi_description.py — 读取 VI Description（通过 COM）

工作原理

1. VI 文件头解析 — 自动识别保存版本

读取 VI 文件头前 512 字节，查找 00 00 00 A0 标记， 随后两个字节以 BCD 格式编码主次版本号。例如：

... 00 00 00 A0 17 00 ...
                ^^  主版本 BCD = 0x17 → 23（十进制）？
                    实为 LabVIEW 2017 存储方式：高4位/低4位各表示一位十进制

无需安装 LabVIEW 或调用 COM，即可在毫秒级得到保存版本。

2. 注册表扫描 — 枚举已安装版本

同时读取 32 位和 64 位注册表视图：

HKLM\SOFTWARE\National Instruments\LabVIEW\<major.minor>\Path

对每条路径读取 LabVIEW.exe 的 PE Machine 字段，判断 x86 / x64。

3. 位数匹配的脚本宿主

LabVIEW COM 只能被相同位数的进程正常访问：

目标 LabVIEW使用的 cscript.exe
x86（32 位）C:\Windows\SysWOW64\cscript.exe
x64（64 位）C:\Windows\System32\cscript.exe

Python 本身的位数不影响结果——正确的宿主由 LabVIEW 安装决定。

4. VBScript Worker 连接流程

  1. 清理目标 LabVIEW.exe /Automation 残留进程。
  2. 以 /Automation 启动目标 LabVIEW.exe。
  3. 循环调用 CreateObject("LabVIEW.Application")， 核对 ApplicationDirectory 与 Version 是否匹配。
  4. 命中后调用 app.GetVIReference(viPath)， 读取 vi.Description。
  5. 结果以 Base64-UTF-8 编码写入临时响应文件，Python 解码后输出。
注意：不会静默回退到无关的活动 COM 实例。 若目标版本不匹配则直接报错，避免读到错误版本的缓存属性。

快速开始

命令行

# 自动按 VI 保存版本选择目标 LabVIEW
python scripts/read_vi_description.py "C:\path\to\test.vi"

# 显式指定版本
python scripts/read_vi_description.py --labview-version 2017 "C:\path\to\test.vi"

# 同版本双位数并存时，显式指定位数
python scripts/read_vi_description.py --labview-version 2025 --labview-bitness x64 "C:\path\to\test.vi"
python scripts/read_vi_description.py --labview-version 2025 --labview-bitness x86 "C:\path\to\test.vi"

# 输出版本选择与命中诊断（写入 stderr）
python scripts/read_vi_description.py --verbose "C:\path\to\test.vi"

Python API

from scripts.read_vi_description import read_description

# 自动版本
desc = read_description(r"C:\path\to\test.vi")

# 显式版本
desc = read_description(r"C:\path\to\test.vi", labview_version="2017")

# 显式版本 + 位数
desc = read_description(r"C:\path\to\test.vi", labview_version=2025, labview_bitness="x64")

print(desc)

参数说明

参数类型说明
vi_pathstrVI 文件路径（绝对或相对）
labview_versionstr / int / tuple 目标 LabVIEW 版本，支持 2017、17.0、2025.3、(17,0)。省略时自动识别。
labview_bitnessstr 目标位数：x86、x64、32、64。省略时优先与当前 Python 进程位数一致。
--verboseflag （仅命令行）把版本选择与命中诊断输出到 stderr。

退出码

退出码含义
0成功，Description 输出到 stdout
2VI 文件不存在
3连接 LabVIEW 失败或读取属性失败
4参数错误或其他未预期错误

依赖

  • Python 3.8+（仅标准库，无第三方依赖）
  • Windows 系统（需要 cscript.exe、WScript、ADODB、Msxml2）
  • 目标版本的 LabVIEW 已安装且注册了 COM 服务器

已知边界

  • 仅支持 Windows 平台。
  • 仅读取 VI.Description，不做写操作。
  • VI 文件头可以提供保存版本，但不包含 x86/x64 信息；同版本双位数并存时，请显式传 --labview-bitness。
  • 若目标 LabVIEW 未安装，脚本直接报错（不静默连接错误版本）。

read_vi_props.py — 读取 VI 全部属性（通过 COM）

在 read_vi_description.py 的基础上扩展，使用相同的版本识别、COM 连接和位数匹配机制， 能够一次性读取 VI 的所有 COM 可访问属性。 单个属性读取失败（例如属性在该版本中不存在）不会导致整体失败，所有其他属性仍会正常返回。

命令行用法

# 读取全部属性，JSON 输出（默认）
python scripts/read_vi_props.py "C:\path\to\test.vi"

# 人类可读文本输出
python scripts/read_vi_props.py --format text "C:\path\to\test.vi"

# 只读取指定属性
python scripts/read_vi_props.py --props Name,Description,Priority "C:\path\to\test.vi"

# 显式指定 LabVIEW 版本和位数
python scripts/read_vi_props.py --labview-version 2025 --labview-bitness x64 "C:\path\to\test.vi"

# 输出版本选择与命中诊断（写入 stderr）
python scripts/read_vi_props.py --verbose "C:\path\to\test.vi"

Python API

from scripts.read_vi_props import read_vi_props

# 读取全部属性
props = read_vi_props(r"C:\path\to\test.vi")

# 读取指定属性
props = read_vi_props(
    r"C:\path\to\test.vi",
    props=["Name", "Description", "Priority"],
)

# 显式版本 + 位数
props = read_vi_props(
    r"C:\path\to\test.vi",
    labview_version="2025",
    labview_bitness="x64",
)

# 访问单个属性
if props["Description"]["ok"]:
    print(props["Description"]["value"])
else:
    print("读取失败:", props["Description"]["error"])

退出码

退出码含义
0成功，属性输出到 stdout
2VI 文件不存在
3连接 LabVIEW 失败或读取属性失败
4参数错误或未知属性名

已知边界

  • 仅支持 Windows 平台。
  • 部分属性（InlineSubVI、Scalable）在较旧 LabVIEW 版本中不存在；单属性失败时 ok=false，不影响其他属性。
  • 同版本 x86/x64 并存时，请显式传 --labview-bitness。
  • 所有属性值均以字符串形式返回（包括 Boolean 和 Number），调用方需自行转换类型。
  • 不做写操作，属性表中标注 R/W 的属性本版本仅支持读取。

read_vi_icon.py — 导出 VI 图标（纯文件解析）

该脚本不依赖 LabVIEW、COM 或第三方工具，直接解析 VI 的 RSRC 资源表，提取图标资源并输出为 PNG。

适用场景

  • 批量提取 VI 图标。
  • CI / 服务器环境下无 LabVIEW 安装时仍可运行。
  • 希望避免 COM 连接、宿主位数匹配等环境约束。

命令行用法

# 单文件导出
python scripts/read_vi_icon.py "C:\path\to\test.vi"

# 指定输出路径
python scripts/read_vi_icon.py "C:\path\to\test.vi" --output "C:\path\to\icon.png"

# 放大倍数
python scripts/read_vi_icon.py "C:\path\to\test.vi" --scale 8

# 递归扫描目录并批量输出
python scripts/read_vi_icon.py "C:\path\to\vi-dir" --recursive

# JSON 输出
python scripts/read_vi_icon.py "C:\path\to\test.vi" --json

输出说明

  • 默认输出 PNG。
  • 优先读取彩色图标资源，缺失时回退到单色图标资源。
  • 对当前样例 2017.vi、2020.vi、2025.vi 已成功导出图标。

依赖

  • Python 3.8+。
  • 仅标准库，无第三方依赖。

已知边界

  • 当前目标是稳定提取常见 VI 图标资源，不覆盖所有潜在 RSRC 变体。
  • 若某些 VI 使用非常规图标资源布局，可能需要继续扩展资源解析分支。

save_vi_panel_image.py — 导出前面板 / 背面板图片

该脚本使用 Python 直接调用 LabVIEW VI Server，对应 Invoke Node 路线，不使用屏幕截图，也不要求把前面板显示到桌面。

工作方式

目标实现方式是否显示窗口
前面板（fp）OpenFrontPanel(False, Hidden) + GetPanelImage(True, 24)否
背面板（bd）PrintVIToHTML(... Complete ...) 后提取 *d.png否

命令行用法

# 导出前面板
python scripts/save_vi_panel_image.py "C:\path\to\test.vi" --panel fp

# 导出背面板
python scripts/save_vi_panel_image.py "C:\path\to\test.vi" --panel bd

# 指定输出文件
python scripts/save_vi_panel_image.py "C:\path\to\test.vi" --panel fp --output "C:\path\to\panel.png"

# 显式指定 LabVIEW 版本和位数
python scripts/save_vi_panel_image.py "C:\path\to\test.vi" --panel fp --labview-version 2025 --labview-bitness x64

# 输出连接诊断
python scripts/save_vi_panel_image.py "C:\path\to\test.vi" --panel fp --verbose

依赖

  • Windows。
  • 已安装目标版本的 LabVIEW，并注册 COM 服务器。
  • Python 3.8+。
  • pywin32。
  • Pillow。

已验证结果

  • 样例 2025.vi 已成功通过隐藏前面板 + GetPanelImage 导出前面板图片。
  • LabVIEW 2025 自带示例 VI 已成功通过 PrintVIToHTML 导出背面板图片。
  • 样例 2025.vi 的背面板导出会返回“未生成目标面板图像”，与其空白 / 极简 diagram 表现一致。

已知边界

  • 当前纯 Python 直连 COM 方案要求 Python 与目标 LabVIEW 位数一致；例如 x64 Python 只能稳定连接 x64 LabVIEW。
  • 背面板导出依赖 LabVIEW 的 HTML 导出结果。若 VI 的 block diagram 为空白或极简，LabVIEW 可能不会生成 *d.png。
  • 前面板导出虽然不显示窗口，但本质上仍会通过 VI Server 打开目标 VI 的 front panel 对象。

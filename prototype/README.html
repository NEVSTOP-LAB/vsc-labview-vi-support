read_vi_description — Prototype

本 prototype 从 labview-vi-props-skill 中提炼了核心经验，实现了一个专注于跨版本稳定读取 LabVIEW VI Description 的独立脚本对。

文件结构

prototype/
└── scripts/
    ├── read_vi_version.py              # 读取 VI 保存版本（纯文件头，无需 LabVIEW）
    ├── read_vi_description.py          # 读取 VI Description（通过 COM/ActiveX）
    └── read_vi_description_worker.vbs  # VBScript COM Worker

read_vi_version.py — 读取 VI 版本

不依赖 LabVIEW 安装，纯文件头解析，跨平台可用。 适合在没有 LabVIEW 的环境（CI、服务器）中快速批量检查 VI 版本。

用法

# 单文件
python scripts/read_vi_version.py "C:\path\to\test.vi"
# 输出: 17.0

# 完整年份格式
python scripts/read_vi_version.py --year "C:\path\to\test.vi"
# 输出: 2017.0

# 人类可读描述
python scripts/read_vi_version.py --describe "C:\path\to\test.vi"
# 输出: LabVIEW 2017 SP0

# 批量扫描目录（递归）
python scripts/read_vi_version.py --recursive --year "C:\project\src"

# JSON 输出（适合程序消费）
python scripts/read_vi_version.py --json --year "C:\path\to\test.vi"

版本号解码规则

VI 文件头前 512 字节内有 00 00 00 A0 标记，紧随其后两个字节以 BCD 格式编码版本：

字节偏移 +4：主版本 BCD  0x17 → 23（十进制）→ 2017 年
字节偏移 +5：次版本 BCD  0x00 → 0 → SP0，0x03 → 3 → SP3
BCD 字节值十进制主版本完整年份LabVIEW 名称
0x14142014LabVIEW 2014
0x17172017LabVIEW 2017
0x20202020LabVIEW 2020
0x25252025LabVIEW 2025

Python API

from scripts.read_vi_version import read_vi_version, describe_version, major_to_year

version = read_vi_version(r"C:\path\to\test.vi")
# 返回 (17, 0) 或 None（非有效 VI 文件）

if version:
    major, minor = version
    print(major_to_year(major))       # 2017
    print(describe_version(major, minor))  # LabVIEW 2017 SP0

退出码

退出码含义
0全部成功
2文件不存在
3非有效 VI 文件或读取失败

read_vi_description.py — 读取 VI Description（通过 COM）

工作原理

1. VI 文件头解析 — 自动识别保存版本

读取 VI 文件头前 512 字节，查找 00 00 00 A0 标记， 随后两个字节以 BCD 格式编码主次版本号。例如：

... 00 00 00 A0 17 00 ...
                ^^  主版本 BCD = 0x17 → 23（十进制）？
                    实为 LabVIEW 2017 存储方式：高4位/低4位各表示一位十进制

无需安装 LabVIEW 或调用 COM，即可在毫秒级得到保存版本。

2. 注册表扫描 — 枚举已安装版本

同时读取 32 位和 64 位注册表视图：

HKLM\SOFTWARE\National Instruments\LabVIEW\<major.minor>\Path

对每条路径读取 LabVIEW.exe 的 PE Machine 字段，判断 x86 / x64。

3. 位数匹配的脚本宿主

LabVIEW COM 只能被相同位数的进程正常访问：

目标 LabVIEW使用的 cscript.exe
x86（32 位）C:\Windows\SysWOW64\cscript.exe
x64（64 位）C:\Windows\System32\cscript.exe

Python 本身的位数不影响结果——正确的宿主由 LabVIEW 安装决定。

4. VBScript Worker 连接流程

  1. 清理目标 LabVIEW.exe /Automation 残留进程。
  2. 以 /Automation 启动目标 LabVIEW.exe。
  3. 循环调用 CreateObject("LabVIEW.Application")， 核对 ApplicationDirectory 与 Version 是否匹配。
  4. 命中后调用 app.GetVIReference(viPath)， 读取 vi.Description。
  5. 结果以 Base64-UTF-8 编码写入临时响应文件，Python 解码后输出。
注意：不会静默回退到无关的活动 COM 实例。 若目标版本不匹配则直接报错，避免读到错误版本的缓存属性。

快速开始

命令行

# 自动按 VI 保存版本选择目标 LabVIEW
python scripts/read_vi_description.py "C:\path\to\test.vi"

# 显式指定版本
python scripts/read_vi_description.py --labview-version 2017 "C:\path\to\test.vi"

# 同版本双位数并存时，显式指定位数
python scripts/read_vi_description.py --labview-version 2025 --labview-bitness x64 "C:\path\to\test.vi"
python scripts/read_vi_description.py --labview-version 2025 --labview-bitness x86 "C:\path\to\test.vi"

# 输出版本选择与命中诊断（写入 stderr）
python scripts/read_vi_description.py --verbose "C:\path\to\test.vi"

Python API

from scripts.read_vi_description import read_description

# 自动版本
desc = read_description(r"C:\path\to\test.vi")

# 显式版本
desc = read_description(r"C:\path\to\test.vi", labview_version="2017")

# 显式版本 + 位数
desc = read_description(r"C:\path\to\test.vi", labview_version=2025, labview_bitness="x64")

print(desc)

参数说明

参数类型说明
vi_pathstrVI 文件路径（绝对或相对）
labview_versionstr / int / tuple 目标 LabVIEW 版本，支持 2017、17.0、2025.3、(17,0)。省略时自动识别。
labview_bitnessstr 目标位数：x86、x64、32、64。省略时优先与当前 Python 进程位数一致。
--verboseflag （仅命令行）把版本选择与命中诊断输出到 stderr。

退出码

退出码含义
0成功，Description 输出到 stdout
2VI 文件不存在
3连接 LabVIEW 失败或读取属性失败
4参数错误或其他未预期错误

依赖

  • Python 3.8+（仅标准库，无第三方依赖）
  • Windows 系统（需要 cscript.exe、WScript、ADODB、Msxml2）
  • 目标版本的 LabVIEW 已安装且注册了 COM 服务器

已知边界

  • 仅支持 Windows 平台。
  • 仅读取 VI.Description，不做写操作。
  • VI 文件头可以提供保存版本，但不包含 x86/x64 信息；同版本双位数并存时，请显式传 --labview-bitness。
  • 若目标 LabVIEW 未安装，脚本直接报错（不静默连接错误版本）。
